Report Page Filter.

The scanner offers an extensive filtering option to narrow and maneuvre through the hordes of data to find the desired subset.

Clicking the button, a pop-up will appear for the user to specify the data they want to fetch.


Following is a brief explanation of the various filtering options
Email Date Range: Items received/created within a date-range.

Content-type: Where the content should be located

Accounts: Whose account(s) to search

Account contains: to filter content in accounts based on a feature, for instance, their @domain (Sixtus-compliance.dk find GDPR-data in all accounts with @sixtus-compliance.dk in their email.)

Senders: Based off of total list of email-senders within the accounts a user is able to interact with. A super-user list will contain a senders list of all senders of the total tenant as they have access to all accounts.

Sender-domain: A quicker way to narrow down senders by groups

Clear filter: Will Clear any and all filters inserted to a completely clear filter.

Reset filter will assume a standard filter (which is by default set to include the entirety of the userâ€™s accessibility).

Restore to Current Filter will restore any filters most recently inserted by then cleared.

GDPR Classifications will allow users quickly to filter what type of GDPR-data they want to browse in their findings. The three-structure is interactive and may assume both Categories and Areas.



Usage and Example

Make sure to correctly clear the "Sender" or "Account" filters properly if a single email is to be selected. The filter will make a too large fetch if, for instance, the "Sender Domain" is full of entries while the "Sender" contains only a single account if the purpose was to find the single account.
